Module libErrorFunctions

Error handling library.

Provides standardised and threaded functions which push error messages to the mission dialogue.

Error Handler

ThrowObjectInitialisationError (initObj) Throws an error message.
ThrowObjectAlreadyRegisteredError (objectType, objectName) Throws an error message.
ThrowObjectNotFoundError (missingObject) Throws an error if a referenced object could not be found.
ThrowEventNotRegisteredError () Throws an error if an event type doesn't exist or could not register.
ThrowThreadCrashedError (thread) Throws an error, if a thread crashed.

Error Handler

ThrowObjectInitialisationError (initObj)
Throws an error message. Thrown if for some reason a object could not be initialised. Wrapper for threaded call. A restart of the Campaign is advised.

Parameters:

  • initObj String MasterTextFile key for the object which failed to initialise.

See also:

    OutObjectInitialisationError
ThrowObjectAlreadyRegisteredError (objectType, objectName)
Throws an error message. Thrown if the game tries to register an already existing object. As long as it's not an initialisation error, things should be fine. Wrapper for threaded call.

Parameters:

  • objectType String MasterTextFile key for the object type which failed to initialise.
  • objectName String MasterTextFile key for the object name which failed to initialise.

See also:

    OutObjectAlreadyRegisteredError
ThrowObjectNotFoundError (missingObject)
Throws an error if a referenced object could not be found. This is a critical error which will usually mess up data handling, and might crash the script. Wrapper for threaded call.

Parameters:

  • missingObject String The missing object's MasterTextFile key.

See also:

    OutObjectNotFoundError
ThrowEventNotRegisteredError ()
Throws an error if an event type doesn't exist or could not register. Wrapper for threaded call.

See also:

    OutEventNotRegisteredError
ThrowThreadCrashedError (thread)
Throws an error, if a thread crashed.

Parameters:

  • thread string
